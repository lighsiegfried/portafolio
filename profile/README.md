## Quick highlights

- **Full-stack, cloud-native** — React 18 + Vite + Three.js SPA backed by a serverless AWS API (Lambda, API Gateway, DynamoDB).
- **Infrastructure as code & secure CI/CD** — Terraform modules + GitHub Actions with **OIDC (no static AWS keys)**.
- **Native Android** — offline-first apps in Kotlin & Jetpack Compose with encrypted local storage.
- **Local-first AI** — a Windows assistant that runs models locally via Ollama, no cloud AI APIs.
- **Engineering discipline** — structured logging, CloudWatch alarms, **246 backend tests**, and documentation treated as part of the product.

### 📱 Mis Finanzas — Android local-first
Native Android personal-finance app: **offline-first**, encrypted on-device storage, Clean Architecture + MVVM, portable encrypted backups and a read-only local assistant. No cloud sync.

`Kotlin` · `Jetpack Compose` · `Room` · `SQLCipher` · `Android`

[**Code**](https://github.com/lighsiegfried/Finanzas)

A voice/text assistant for Windows that plans with a **local LLM (Ollama)** but never lets the model act directly — every action passes a central safety layer, sensitive actions need confirmation, destructive ones stay blocked.

`Python` · `FastAPI` · `Ollama` · `SQLite` · `Windows`

## How I work

- **Architecture first** — clear separation of concerns (Clean Architecture / MVVM, repository abstractions, own Lambda router) so business logic never lives in the UI.
- **Infrastructure as code** — reproducible environments with Terraform; nothing clicked by hand in the console.
- **Testing & reliability** — automated tests on rejection and failure paths, atomic writes to protect data integrity, structured logs and CloudWatch alarms.
- **Secure CI/CD** — GitHub Actions with OIDC (no long-lived cloud keys) and smoke tests against production.
- **Local-first & privacy** — encrypted on-device storage and local model execution when the use case calls for it; no unnecessary cloud dependencies.
- **Documentation as a deliverable** — READMEs, diagrams and decision notes so the work can be understood and audited.
